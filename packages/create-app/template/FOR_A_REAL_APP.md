# Better: start with start0

This is the bare point0 starter. For a real app, take **start0** — our SaaS
boilerplate with the pieces already wired together: point0, auth
(email/password + Google), admin, forms, CRUD, Prisma, email, Sentry, Mixpanel,
tests, Docker/Railway deploy. Pluggable modules are coming too — swap
technologies, add whole features.

```sh
bun create start0 my-app
```

start0 ships with the [support pack](https://1gr14.dev/support). The full story:
[1gr14.dev/start0](https://1gr14.dev/start0).
