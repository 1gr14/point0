# Error logging loses stacks / prints "[ undefined ]" — incident, resolved by design

**Status:** RESOLVED 2026-06-11 (design fix shipped; one external item left) · **Area:** error0 / point0 core logger / app loggers (start0, site) ·
**Origin:** 2026-06-10, the 1gr14 site crashed at boot and the log was useless for diagnosis.

## What happened (short)

better-auth 1.6.16's `deviceAuthorization` plugin threw a zod v4 `ZodError` during init. The log showed
no stack, a brace-mangled message (`[ undefined ]`), no error name. Three mechanisms: (1) Bun wipes
frames on `Error.captureStackTrace(e, callee)` (zod v4 `.parse()`); (2) LogTape parsed the raw message
as a template; (3) one `serialize(isPublic)` conflated the WIRE format with the LOG format, so
publicity/env gating stripped exactly what the operator needed.

## How it was resolved (2026-06-11)

The design fix: two named serialization audiences instead of one gated serialize.

- **error0 0.3.0 (released):** `serializePublic()` / `serializePrivate()` (sugar over
  `serialize(isPublic)`); `name` always in output; string mark → instance name; plugin option
  `transport: 'public' | 'private' | 'none'` (replaces plugin-level `isPublic`; expectedPlugin's
  inverted gate fixed); `causePlugin` now serializes foreign Error causes ({name, message, stack},
  recursive, cycle/depth-guarded) — a ZodError cause survives `serializePrivate()`.
- **point0 (this repo, with the matching change set):** `ClassLikeError0` = `from` +
  `serializePublic` + `serializePrivate` (no plain `serialize`); `ErrorPoint0` implements both,
  `toJSON` → public (accidental-stringify safety net); `_defaultLogFn` json mode ignores `toJSON`,
  uses `serializePrivate()` with identity backfill (name/message/stack fill-if-absent), walks the
  cause chain for foreign errors; the wire call-sites (fetcher `toJsonErrorResponse`, dehydrated
  state, DefaultErrorComponent) pick public-in-prod / private-in-dev at the call site — no more
  NODE_ENV inside serializers.
- **start0 + site:** AppError on the new prescription (transport options, `causePlugin()` added,
  stack override gates by `isPublic`, not env); logger feeds props from `AppError.serializePrivate`;
  stack rides the message in pretty mode only — json mode carries it structurally. LogTape brace
  escaping kept. error0 ^0.3.0 installed; fresh point0 tarballs installed. Verified by replaying the
  incident (AppError + ZodError-subclass cause) in both LOG_MODE values + boot smoke (HTTP 200).

## Debugging rule of thumb (kept from the incident)

A frameless error in a Bun app usually never HAD frames (zod `.parse()` / captureStackTrace) — suspect
the throw site, not the logger. Root cause per upstream (oven-sh/bun#13904): JSC tail calls eliminate
the callee's frame, then Bun's not-found fallback drops the whole trace. To get frames back while
debugging, rerun with `BUN_JSC_useTailCalls=0` (deliberately NOT baked into the apps — dev opt-in only).
`undefined` inside logged braces means raw text hit a LogTape template.
